# 科科命理宇宙站｜姓名學筆畫規則與資料表規格 v0.7.4

## 版本定位

v0.7.4 是姓名學第二核心候選的規則邊界文件版。本版只定義未來姓名學正式計算前需要確認的筆畫規則、資料表欄位、授權來源、fallback 政策與測試案例，不新增 calculator，不新增 meaning data，也不把姓名學標示為 calculated。

v0.7.5 新增 `NAME_STROKE_SEED_PLAN.md`。seed 規劃文件是本規則文件之後的資料表前置文件，用來規劃未來 seed 欄位、第一批測試字元、字元狀態、fallback 與測試分組。v0.7.5 仍不新增正式筆畫資料表 JS。

## 目前狀態

- 模組：姓名學
- 狀態：planning
- 計算狀態：not-calculated
- 角色：second core candidate
- 目前資料來源：`coreInputProfile.personal.fullName`

- 本版不提供改名建議。

- 本版不宣稱姓名學資料已完整。

## 規則決策清單

正式接入姓名學前，至少需要先決定：

1. 採用康熙筆畫或現代筆畫。
2. 筆畫資料表來源與授權。
3. 繁體字、簡體字與轉換政策。
4. 異體字歸併規則。
5. 缺字 fallback 與人工校對流程。
6. 複姓與單姓處理方式。
7. 單名、雙名與長名字處理方式。
8. 五格算法版本。
9. 三才五行規則版本。
10. 字音、字義、字形是否只作參考資訊。
11. 測試案例與人工驗收標準。
12. 未來 calculator placeholder 的輸入與輸出邊界。
13. 未來 meaning data 的資料來源與安全線。

## 筆畫資料表欄位規格

未來若建立姓名學筆畫 seed 資料表，建議每筆至少包含：

| 欄位 | 用途 |
| --- | --- |
| `char` | 原始字元 |
| `kangxiStroke` | 康熙筆畫，尚未選定前可為 null |
| `modernStroke` | 現代筆畫，尚未選定前可為 null |
| `normalizedChar` | 正規化後字元 |
| `variantGroup` | 異體字群組 |
| `source` | 筆畫來源 |
| `sourceLicense` | 來源授權 |
| `sourceNote` | 來源備註 |
| `status` | seed / reviewed / missing / fallback |
| `fallbackPolicy` | 缺字或不確定字處理方式 |
| `notes` | 人工校對備註 |

## 資料表授權邊界

姓名學筆畫資料表不可直接複製來源不明或授權不清的資料。正式建立資料表前需確認：

- 資料來源是否可用於本專案。
- sourceLicense 是否能清楚記錄。
- 是否需要人工校對。
- 是否允許公開在 GitHub Pages 靜態 repo。
- 是否可在未來拆成 seed data 或人工維護表。

## fallback 規則

在資料表不完整或字元無法命中時，建議採用保守策略：

- 顯示「本字尚未收錄」或「需要人工校對」。
- 不自行猜測筆畫。
- 不輸出正式五格結果。
- 不提供改名建議。
- 不宣稱資料完整。

## 測試案例規劃

未來姓名學接入前至少需要準備：

- 科科 seed 名稱。
- 單姓單名。
- 單姓雙名。
- 複姓測試。
- 空姓氏或暱稱測試。
- 異體字測試。
- 缺字 fallback 測試。
- 筆畫資料表命中測試。
- 五格欄位輸出測試。
- 安全線顯示測試。
